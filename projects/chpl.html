CHPL Redesign

Overview

As a kid, I’d spend hours and hours in my local library—Clarendon Hills Public Library. My time at the library was crucial for developing who I am today, so I wanted to pay that investment forward.

In this project, I analyzed, redesigned, and implemented a responsive version of CHPL's website. My goal was to identify issues with the site's conceptual model, then correct them to make it intuitive to users. I focused on implementing good hierarchy to make the site easy to parse and follow, a consistent style guide for learnability and memorability, and abundant accessibility features.

I hope my redesign can encourage others to visit, so they can discover the same resources and worlds I did :).

Problem

How can we redesign a website to be user-friendly, accessible, and responsive?

Original Site

I began by conducting an analysis of the original site's issues. Immediately, what stuck out to me the most was how egregiously cluttered the navbar was—definitely something that needed reworking. Besides that, the main issues I noticed were an unclear visual hierarchy and inconsistent color/style, impacting learnability and memorability.

Usability Problems

Efficiency Issues

  1. Location and structure of sections/headings is inconsistent—this can be confusing for users and cause slowdowns when scanning.
  2. No clear hierarchy in the navbar makes it difficult for users to quickly find the link they're looking for.

Learnability Issues

  1. Buttons and links have extremely varied style and wording, which can confuse first-time users about the purpose of the component and the result of any action they may take.
  2. Having so many different font styles make it hard to determine what's important on the page, especially for first-time users.

Memorability Issues

  1. Too many navigation paths to the same content (e.g., multiple ways to access “Library Hours” and “My Account”) make it hard for returning users to remember the best route.
  2. The abundance of colors makes the page visually interesting, but also makes it hard to remember the purpose of each color and how it relates to the content.

Accessibility Issues

  1. Extremely unintuitive navigation order.
  2. Through but often redundant alternative text might be monotonous for screen reader users.

Visual Redesign

Before starting my redesign, I constructed a style guide based on the original website, giving each different component and color a purpose to enhance learnability and memorability. I then created hi-fi mockups on Figma to visualize the responsiveness of my webpage for three different screen sizes.

Style Guide

The style guide I created is a comprehensive reference for my redesign's design system, including typography, color palette, and component styles. It serves as a foundation for the website's visual identity, ensuring consistency across all pages.

Figma Mockups

Using the style guide as a reference, I created hi-fi mockups of the website in Figma. The mockups showcase the responsive design for three different screen sizes: desktop, tablet, and mobile.

Key Changes

  • For learnability & memorability, buttons have different styles depending solely on their function: one primary button, one to denote an active tab, and one to denote an inactive tab.
  • Significantly reduced the amount of redundancy on the page by condensing the information to display the essentials. This also aligns more with the conceptual model of the page as a place for users to land and then discover more; not every detail should be on the homepage.
  • Implemented a more distinct visual hierarchy—sections are all stacked in a column, follow the same structure, and cards have consistent formatting.
  • Simplified the navbar, separating sections by general library information, link category, and age demographic to make it easier for users to find the page they're looking for.

Implementation

I used HTML/CSS to build my redesigned website, focusing on responsiveness, efficiency, and simplicity.

Implementation

Established a clearer visual hierarchy using typography and layout, with a consistent structure for all sections and cards to aid in scanning and comprehension.
Consistency Buttons and links vary significantly in style and language, and multiple font styles make it unclear what's most important Created a style guide to standardize buttons (primary, active, inactive) and text elements, enhancing clarity and predictability.
Accessibility Poor navigation order and unclear alt text usage. Improved navigation order for screen readers and ensured alt text is concise and relevant to the image's context.
